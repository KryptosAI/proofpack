ProofPack Help

ProofPack auto-generates dispute evidence for AI SaaS companies. Connect your Stripe account, integrate the SDK, and ProofPack handles chargeback evidence so you don't have to.

1. Getting Started

  1. Create an account at proofpack.dev.
  2. Grab your API key from the dashboard under Settings → API Keys. Keys start with ppk_.
  3. Install the SDK for your stack.
npm install @proofpack/sdk
pip install proofpack

2. Integrating the SDK

Once installed, initialize ProofPack and start tracking user behavior. The SDK captures product usage data that ProofPack uses to build compelling dispute evidence showing legitimate customer activity.

JavaScript / TypeScript

import { init } from '@proofpack/sdk';

const pp = init({ apiKey: 'ppk_live_abc123...' });

pp.track({
  userId: 'user_abc',
  event: 'output.generated',
  metadata: { model: 'gpt-4', tokens: 512 }
});

Python

from proofpack import ProofPack, ProofEventPayload

pp = ProofPack(api_key="ppk_live_abc123...")

pp.track(ProofEventPayload(
    user_id="user_abc",
    event="output.generated",
    metadata={"model": "gpt-4", "tokens": 512}
))

Events to track

Track meaningful product actions. ProofPack uses these to prove the customer was actively using your service:

  • output.generated — AI output was generated
  • session.started — User logged in or began a session
  • feature.used — A specific feature was accessed
  • plan.upgraded — User changed plans
  • email.verified — User confirmed their identity
  • ip.address — Geo / IP data for the customer

Server-side tracking (optional)

You can also send events from your backend using the REST API:

curl -X POST https://api.proofpack.dev/v1/events \
  -H "Authorization: Bearer ppk_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "user_abc",
    "event": "output.generated",
    "metadata": { "model": "gpt-4", "tokens": 512 }
  }'

3. Connecting Stripe

ProofPack needs read access to your Stripe disputes so it can generate evidence and (optionally) submit it on your behalf.

  1. Go to Settings → Integrations in the ProofPack dashboard.
  2. Click Connect Stripe. You'll be redirected to Stripe's OAuth flow.
  3. Select the Stripe account you use for this product and authorize the connection.
  4. After redirect, ProofPack will begin syncing your dispute data automatically.

ProofPack requests the following Stripe scopes:

  • read_write — Required for disputes (read + submit evidence)
  • read_only — For customers, charges, and payment intents

No other data is accessed. You can revoke access at any time from your Stripe dashboard.

4. Viewing & Managing Disputes

Once connected, all open and recently closed disputes appear on your Disputes page. Each row shows:

  • Dispute status (needs response, won, lost, under review)
  • Dispute amount and currency
  • Reason code (fraudulent, product not received, etc.)
  • Due date for evidence submission
  • Customer ID and associated usage events

Click any dispute to drill into the detail view, which shows the full timeline of customer activity tracked by the SDK.

5. Generating Evidence PDFs

ProofPack compiles SDK-tracked events, Stripe charge metadata, and your company profile into a single, ready-to-submit evidence PDF.

To generate evidence for a dispute:

  1. Open the dispute from the Disputes list.
  2. Review the auto-populated evidence summary. ProofPack includes:
    • Timeline of customer events and API usage
    • IP addresses, timestamps, and user-agent strings
    • Terms of service acceptance and billing history
    • Product/service description from your ProofPack profile
  3. Add any custom notes or attachments if needed.
  4. Click Generate PDF. ProofPack produces a Stripe-compliant evidence document.

You can also trigger generation via the API:

curl -X POST https://api.proofpack.dev/v1/disputes/dp_xxx/evidence \
  -H "Authorization: Bearer ppk_live_abc123..."

6. Submitting Evidence to Stripe

After reviewing the generated evidence, you have two options:

  • Manual submit — Download the PDF and upload it to Stripe yourself.
  • Auto-submit — Enable "Auto-submit evidence" in Settings. ProofPack will file evidence automatically 24 hours before the Stripe deadline (you'll receive an email 48 hours before for a final review window).

To submit via API:

curl -X POST https://api.proofpack.dev/v1/disputes/dp_xxx/submit \
  -H "Authorization: Bearer ppk_live_abc123..."

7. FAQ

What is a chargeback / dispute?

A chargeback occurs when a cardholder disputes a charge with their bank. Stripe deducts the amount plus a fee from your account and opens a dispute. You have a limited window to submit evidence proving the charge was legitimate.

How does ProofPack help win disputes?

ProofPack correlates your SDK-tracked usage events with Stripe charge data to build a timeline of legitimate customer activity. The resulting evidence shows the customer actively used and benefited from your service, which is exactly what card networks look for.

What Stripe evidence requirements does ProofPack cover?

ProofPack addresses the most common Stripe evidence categories: service documentation, access/activity logs, customer communication, terms of service acceptance, and billing history. For specific reason codes (e.g., "product not received"), ProofPack tailors the evidence package accordingly.

What if the customer never used the product?

ProofPack still builds a case using billing records, IP logs, sign-up timestamps, and your terms of service. Even minimal activity can be enough when the service was made available and accessible.

Is my data secure?

Yes. ProofPack encrypts all data in transit (TLS 1.3) and at rest (AES-256). API keys are hashed. Stripe credentials are handled via OAuth and never stored on our servers. We are SOC 2 Type II compliant.

Do I need to track every user action?

No. Track meaningful product events — logins, outputs generated, features used. A handful of high-signal events per session is enough. The more context you provide, the stronger the evidence.

Can I use ProofPack without the SDK?

Yes. You can send events through the REST API or import historical data via CSV upload. The SDK is just the most convenient path.

What does it cost?

ProofPack is free for up to 5 disputes per month. Paid plans start at $49/month and scale with dispute volume. See our pricing page for details.

How do I disconnect Stripe?

Go to Settings → Integrations and click Disconnect. You can also revoke access from your Stripe dashboard under Settings → Connected applications.
